🚀 快速开始

发送第一条消息，理解基本概念，配置常用功能

⏱️ 学习时长：30 分钟  |  ⭐ 难度等级：简单  |  ✅ 前置要求：已完成安装

🎯 学习目标

完成本教程后，你将能够：

  • ✅ 发送第一条消息并收到 AI 回复
  • ✅ 理解 OpenClaw 的基本概念（Session、Agent、Skill）
  • ✅ 配置常用功能（模型、提示词、渠道）
  • ✅ 排查常见问题

📋 前置条件

  • ✅ 已完成安装指南
  • ✅ Gateway 正在运行
  • ✅ 至少一个渠道已连接（Telegram/微信/Discord）

步骤 1: 验证连接

首先检查 Gateway 是否正常运行：

curl http://localhost:3000/health # 预期输出 {"status":"ok","uptime":12345}

步骤 2: 发送第一条消息

Telegram

  1. 在 Telegram 中找到您的机器人
  2. 发送消息：你好
  3. 等待 AI 回复

预期回复：你好！我是你的 OpenClaw 助手，有什么可以帮你的吗？

微信

  1. 在微信中找到您的机器人
  2. 发送消息：测试
  3. 等待 AI 回复

Discord

  1. 在服务器中找到机器人
  2. @机器人并发送：你好
  3. 等待 AI 回复

步骤 3: 理解基本概念

Session（会话）

Session 是对话的上下文容器。每次对话都会创建一个 Session，保存历史消息和状态。

  • 作用：保持对话连贯性
  • 生命周期：默认 30 分钟无活动后过期
  • 存储：MEMORY.md 和 memory/ 目录

Agent（代理）

Agent 是 AI 的大脑，负责理解消息、调用工具、生成回复。

  • 模型：支持 GPT-4、Claude、通义千问等
  • 提示词：定义 AI 的角色和行为
  • 工具：可以调用 Skills 执行特定任务

Skill（技能）

Skill 是扩展功能的插件，让 AI 能够执行特定任务。

  • 示例：天气查询、文件处理、定时任务
  • 触发：根据用户消息自动触发
  • 开发：可以自定义 Skills

步骤 4: 配置常用功能

修改系统提示词

在 config.yaml 中修改 systemPrompt：

systemPrompt: | 你是一位专业、友好、高效的 AI 助手。 你擅长用简洁清晰的语言解答问题。 你会主动询问不明确的需求。 你乐于提供实用的建议和解决方案。

切换 AI 模型

在 config.yaml 中修改 model：

channels: telegram: - token: "your-bot-token" model: "qwen-plus" # 或 gpt-4, claude-3 thinking: "on" # 开启推理模式

配置记忆系统

在 MEMORY.md 中记录重要信息：

# MEMORY.md ## 用户偏好 - 称呼：陈辰 - 时区：Asia/Shanghai - 语言：中文 ## 项目上下文 - 当前项目：OpenClaw 知识库 - 目标：建设完整的教程体系

步骤 5: 测试常用功能

测试记忆功能

发送消息：记住我喜欢喝美式咖啡

稍后询问：我喜欢喝什么？

预期回复：你喜欢喝美式咖啡。

测试 Skills

发送消息：今天天气怎么样？

如果配置了 weather Skill，AI 会自动查询并回复天气信息。

测试定时任务

发送消息：明天早上 9 点提醒我开会

AI 会创建定时任务，并在指定时间发送提醒。

🐛 常见问题

Q1: 机器人无响应

检查项：

  • Gateway 是否正常运行
  • 渠道配置是否正确（Bot Token）
  • 网络连接是否正常
  • 查看 Gateway 日志是否有错误

Q2: AI 回复很慢

可能原因：

  • 模型推理时间较长（特别是 thinking: on）
  • 网络连接不稳定
  • API 服务响应慢

解决方案：尝试切换到更快的模型，或关闭 thinking 模式。

Q3: 记忆不生效

检查项：

  • MEMORY.md 文件是否存在
  • 是否在群聊中（群聊不加载个人记忆）
  • memory_search 是否正常工作
