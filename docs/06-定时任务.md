# ⏰ 定时任务配置 - 让 AI 自动工作

> 配置 Cron Jobs，让 OpenClaw 按时执行任务

## 🎯 什么是定时任务？

定时任务 = 在指定时间自动执行某些操作

**使用场景**:
- ⏰ 每天早上 8 点发送天气提醒
- 📅 每周一上午 9 点生成周报
- 🔔 每 10 分钟检查一次邮件
- 📊 每天凌晨 2 点备份数据

## 📋 Cron 任务结构

```json
{
  "name": "任务名称",
  "schedule": { ... },      // 何时执行
  "payload": { ... },       // 执行什么
  "sessionTarget": "main",  // 发送到哪个会话
  "enabled": true           // 是否启用
}
```

## 🕐 Schedule 类型（3 种）

### 1. 一次性任务 (`at`)

在特定时间点执行一次

```json
{
  "kind": "at",
  "atMs": 1772938652000  // Unix 时间戳（毫秒）
}
```

**使用场景**:
- 提醒开会
- 一次性通知
- 定时发送消息

**示例**: 明天上午 10 点提醒
```javascript
const tomorrow10am = new Date();
tomorrow10am.setDate(tomorrow10am.getDate() + 1);
tomorrow10am.setHours(10, 0, 0, 0);

{
  "kind": "at",
  "atMs": tomorrow10am.getTime()
}
```

### 2. 周期性任务 (`every`)

每隔固定时间执行

```json
{
  "kind": "every",
  "everyMs": 600000,       // 每 10 分钟（毫秒）
  "anchorMs": 1772938652000  // 可选：起始时间
}
```

**常用间隔**:
```
60000     = 1 分钟
300000    = 5 分钟
600000    = 10 分钟
3600000   = 1 小时
86400000  = 24 小时
604800000 = 7 天
```

**使用场景**:
- 进度汇报
- 定期检查
- 心跳检测

### 3. Cron 表达式 (`cron`)

灵活的 cron 语法

```json
{
  "kind": "cron",
  "expr": "0 9 * * 1",  // 每周一 9:00
  "tz": "Asia/Shanghai"  // 时区
}
```

**Cron 语法**:
```
┌───────────── 分钟 (0-59)
│ ┌───────────── 小时 (0-23)
│ │ ┌───────────── 日期 (1-31)
│ │ │ ┌───────────── 月份 (1-12)
│ │ │ │ ┌───────────── 星期 (0-6, 0=周日)
│ │ │ │ │
* * * * *
```

**常用表达式**:
```
# 每天凌晨 2 点
0 2 * * *

# 每 15 分钟
*/15 * * * *

# 工作日 9:00
0 9 * * 1-5

# 每月 1 号
0 0 1 * *

# 每小时
0 * * * *
```

## 📨 Payload 类型（2 种）

### 1. systemEvent（主会话）

向主会话注入系统事件

```json
{
  "kind": "systemEvent",
  "text": "⏰ 提醒：该开会了！"
}
```

**约束**: `sessionTarget` 必须是 `"main"`

**使用场景**:
- 提醒通知
- 状态更新
- 进度汇报

### 2. agentTurn（隔离会话）

在隔离会话中运行 Agent

```json
{
  "kind": "agentTurn",
  "message": "请生成今日新闻摘要",
  "model": "qwen-plus",
  "timeoutSeconds": 300
}
```

**约束**: `sessionTarget` 必须是 `"isolated"`

**使用场景**:
- 自动报告生成
- 后台数据处理
- 独立任务执行

## 🔧 API 操作

### 创建任务
```javascript
await cron({
  action: "add",
  job: { ... }  // 完整任务配置
});
```

### 列出任务
```javascript
const jobs = await cron({
  action: "list",
  includeDisabled: false
});
```

### 更新任务
```javascript
await cron({
  action: "update",
  jobId: "任务 ID",
  patch: {
    enabled: false  // 禁用任务
  }
});
```

### 删除任务
```javascript
await cron({
  action: "remove",
  jobId: "任务 ID"
});
```

### 立即执行
```javascript
await cron({
  action: "run",
  jobId: "任务 ID"
});
```

### 查看执行历史
```javascript
const history = await cron({
  action: "runs",
  jobId: "任务 ID"
});
```

### 发送唤醒事件
```javascript
await cron({
  action: "wake",
  text: "立即检查邮件",
  mode: "now"  // 或 "next-heartbeat"
});
```

## 📊 任务管理最佳实践

### 1. 命名规范
```
✅ daily-weather-report
✅ weekly-summary
✅ progress-check

❌ task1
❌ myjob
❌ test
```

### 2. 错误处理
```javascript
// 在 payload 中包含错误恢复逻辑
{
  "kind": "agentTurn",
  "message": "检查邮件，如果失败则重试",
  "timeoutSeconds": 300
}
```

### 3. 资源控制
```
- 设置合理的 timeout
- 避免过于频繁的执行
- 隔离长时间运行的任务
```

### 4. 监控和日志
```javascript
// 记录每次执行
const log = {
  jobId: "...",
  executedAt: new Date(),
  status: "success/failure",
  duration: 1234  // ms
};
```

## ⚠️ 注意事项

1. **时区问题**: 明确指定 `tz`，避免时区混乱
2. **任务堆积**: 长时间任务用 `isolated` 会话
3. **幂等性**: 确保任务可重复执行
4. **资源限制**: 避免创建过多定时任务
5. **错误恢复**: 考虑失败后的处理策略
