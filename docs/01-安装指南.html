📦 安装指南

零基础安装教程，15 分钟快速部署 OpenClaw 环境

⏱️ 学习时长：15 分钟  |  ⭐ 难度等级：简单  |  ✅ 前置要求：无

🎯 准备工作

在开始之前，请确保您的计算机满足以下基本要求：

  • 操作系统：Windows 10/11、macOS 10.15+、或 Linux
  • 内存：至少 4GB RAM（推荐 8GB）
  • 磁盘空间：至少 1GB 可用空间
  • 网络连接：用于下载依赖和配置渠道

📥 步骤 1：安装 Node.js

OpenClaw 基于 Node.js 构建，首先需要安装 Node.js 运行环境：

  1. 访问 Node.js 官网
  2. 下载 LTS（长期支持）版本
  3. 运行安装程序，按照提示完成安装
  4. 验证安装：打开终端/命令行，输入 node --version
# 验证 Node.js 安装 node --version # 应该显示类似：v20.x.x # 验证 npm 安装 npm --version # 应该显示类似：10.x.x

📦 步骤 2：获取 OpenClaw

有两种方式获取 OpenClaw：

方式 A：从 GitHub 下载（推荐）

# 克隆 OpenClaw 仓库 git clone https://github.com/openclaw/openclaw.git # 进入项目目录 cd openclaw # 安装依赖 npm install

方式 B：直接下载压缩包

  1. 访问 GitHub Releases
  2. 下载最新版本的 zip 文件
  3. 解压到本地目录
  4. 在目录中运行 npm install

⚙️ 步骤 3：配置环境变量

创建 .env 文件并配置必要的密钥：

# 模型 API 密钥（选择一个即可） OPENAI_API_KEY=sk-your-openai-key # 或 ANTHROPIC_API_KEY=sk-ant-your-anthropic-key # 或 DASHSCOPE_API_KEY=sk-your-dashscope-key # 可选：配置其他服务 ELEVENLABS_API_KEY=your-elevenlabs-key # TTS 服务
💡 提示：建议使用 DashScope（阿里云通义千问）以获得更稳定的国内访问体验。

🔧 步骤 4：配置渠道接入

选择您想要使用的通讯渠道进行配置：

Telegram（推荐新手）

  1. 在 Telegram 中搜索 @BotFather
  2. 发送 /newbot 创建新机器人
  3. 按照提示设置机器人名称和用户名
  4. 复制 Bot Token，保存到配置中
# config.yaml 配置示例 channels: telegram: - token: "your-telegram-bot-token" name: "My Assistant" model: "qwen-plus" thinking: "on" systemPrompt: "你是一位专业、友好、高效的 AI 助手..."

🚀 步骤 5：启动 OpenClaw

一切准备就绪！启动 OpenClaw Gateway：

# 启动 Gateway npm start # 或使用 nodemon 进行开发（自动重载） npm run dev

启动成功后，您会看到类似输出：

[Gateway] Starting OpenClaw Gateway... [Channels] Loading channel plugins... [Telegram] Bot "My Assistant" started successfully [Sessions] Session manager initialized [Agents] Agent pool ready [Gateway] Gateway is running! Listening for messages...

✅ 步骤 6：验证安装

  1. 在 Telegram 中找到您的机器人
  2. 发送第一条消息："你好"
  3. 等待 AI 回复（首次可能较慢）
  4. 如果收到回复，说明安装成功！
🎉 恭喜！您已经成功安装并配置了 OpenClaw！
接下来可以查看 快速开始 教程，学习如何使用更多功能。

🐛 常见问题

Q1: 启动时提示 "Cannot find module"

解决方法：运行 npm install 重新安装依赖

Q2: Telegram 机器人无响应

检查项：

  • Bot Token 是否正确配置
  • 网络连接是否正常
  • Gateway 日志是否有错误信息

Q3: API 请求失败

检查项：

  • API Key 是否有效
  • 模型名称是否正确
  • 网络连接是否稳定
