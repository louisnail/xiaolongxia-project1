# 💬 提示词工程 - 让 AI 更懂你

> 好提示词 = 好结果

## 🎯 什么是提示词工程？

提示词工程 = 设计更好的问题，获得更好的答案

**对比**:
```
❌ 普通：帮我写个东西
✅ 优秀：写一个 200 字的产品介绍，突出 3 个核心功能，语气专业但友好
```

## 📐 提示词基础结构

### 标准模板

```markdown
# 角色设定
你是 [角色]，擅长 [技能]

# 任务描述
请完成 [具体任务]

# 背景信息
- 目标用户：[谁]
- 使用场景：[哪里用]
- 关键要点：[重点]

# 输出要求
- 格式：[JSON/列表/段落]
- 长度：[字数限制]
- 风格：[正式/ casual/幽默]

# 示例（可选）
输入：...
输出：...
```

### 实际应用

```markdown
你是资深产品经理，擅长写产品文档

请为 OpenClaw 写一个功能介绍

背景:
- 目标用户：开发者和技术爱好者
- 使用场景：GitHub README
- 核心功能：IM 集成、Skills 扩展、定时任务

要求:
- 格式：Markdown
- 长度：300 字以内
- 风格：专业但易懂
- 包含：3 个核心特性 + 使用示例
```

## 🎨 高级技巧

### 1. 角色扮演

**基础版**:
```
你是一个助手
```

**进阶版**:
```
你是资深 Python 开发者，有 10 年经验，擅长写清晰易懂的代码和详细的注释
```

**效果对比**:
```
基础版回答：print("Hello")

进阶版回答：
# 打印问候语
# 参数：无
# 返回：无
print("Hello World")
```

### 2. 思维链 (Chain of Thought)

**触发词**:
```
请逐步思考...
让我们一步步分析...
先分析，再给出答案...
```

**示例**:
```
问题：小明有 5 个苹果，给了小红 2 个，又买了 3 个，现在有几个？

提示词：请逐步思考，展示计算过程

回答：
1. 初始：5 个苹果
2. 给小红 2 个：5 - 2 = 3 个
3. 又买 3 个：3 + 3 = 6 个
答案：6 个苹果
```

### 3. 少样本学习 (Few-Shot)

**提供示例**:
```
按照以下格式回答：

示例 1:
输入：北京天气
输出：🌤️ 北京：晴，25°C

示例 2:
输入：上海天气  
输出：🌧️ 上海：小雨，22°C

现在回答：
输入：广州天气
输出：
```

### 4. 分隔符使用

**清晰分隔**:
```
请分析以下文本：

"""
这是要分析的文本内容
可以很长
包含多段
"""

请总结主要观点
```

**常用分隔符**:
```
""" 三重引号
--- 分隔线
### 标题
``` 代码块
```

### 5. 明确约束

**模糊**:
```
写短一点
```

**明确**:
```
- 不超过 100 字
- 3 个要点
- 每点不超过 20 字
```

## 🎯 场景化提示词

### 场景 1: 代码生成

```markdown
你是资深前端工程师

任务：创建一个 React 组件

要求:
- 功能：待办事项列表
- 特性：添加、删除、标记完成
- 技术：React Hooks, TypeScript
- 样式：Tailwind CSS
- 包含：完整代码 + 使用示例

输出格式:
1. 组件代码
2. 使用说明
3. 关键逻辑解释
```

### 场景 2: 文档写作

```markdown
你是技术文档专家

任务：编写 API 文档

API 信息:
- 端点：POST /api/users
- 参数：name(string), email(string)
- 返回：用户对象

要求:
- 结构：描述 + 参数 + 示例 + 错误码
- 风格：清晰、简洁、专业
- 包含：curl 示例 + 响应示例
```

### 场景 3: 数据分析

```markdown
你是数据分析师

任务：分析销售数据

数据:
[粘贴数据或描述数据结构]

要求:
1. 计算关键指标（总额、平均、增长率）
2. 识别趋势和异常
3. 给出 3 条 actionable 建议
4. 用表格展示结果
```

### 场景 4: 创意写作

```markdown
你是创意作家

任务：写一个短篇故事

主题：AI 与人类的友谊

要求:
- 长度：500-800 字
- 风格：温暖、感人
- 视角：第一人称
- 包含：开头悬念、中间冲突、结尾升华
```

## 🔧 优化技巧

### 1. 迭代优化

**第 1 版**:
```
写个产品介绍
```

**第 2 版**:
```
写一个 OpenClaw 的产品介绍，面向开发者
```

**第 3 版**:
```
你是技术布道师，写一个 OpenClaw 产品介绍
- 受众：开发者
- 长度：300 字
- 突出：易用性、扩展性
- 包含：代码示例
```

### 2. A/B 测试

```
版本 A: 请总结这篇文章
版本 B: 请用 3 个要点总结这篇文章的核心内容

对比哪个效果更好
```

### 3. 参数调整

```
temperature: 0.3  # 事实性问题
temperature: 0.7  # 创意写作
temperature: 0.9  # 头脑风暴

maxTokens: 100    # 简短回答
maxTokens: 2000   # 详细分析
```

## ⚠️ 常见陷阱

### 1. 太模糊
```
❌ 帮我分析一下
✅ 请分析这个数据的趋势，指出异常值，并给出可能原因
```

### 2. 太多要求
```
❌ 要短、要详细、要专业、要有趣...（矛盾）
✅ 专业风格，300 字，3 个要点
```

### 3. 缺少上下文
```
❌ 这个怎么改？
✅ 这段代码有性能问题，如何优化？[附代码]
```

### 4. 假设 AI 知道
```
❌ 继续昨天的工作
✅ 继续昨天的任务：完成用户登录模块，昨天完成了注册部分...
```

## 📊 提示词检查清单

写完后检查：
- [ ] 角色是否明确？
- [ ] 任务是否具体？
- [ ] 约束是否清晰？
- [ ] 格式是否指定？
- [ ] 示例是否需要？
- [ ] 上下文是否足够？

## 🎓 实战练习

### 练习 1: 改进提示词

原提示词：
```
写个邮件
```

改进后：
```
你是行政助理，帮我写一封会议邀请邮件

信息:
- 会议：Q2 项目评审会
- 时间：下周三 14:00-16:00
- 地点：3 楼会议室
- 参会：项目组全员
- 准备：每人准备 5 分钟汇报

要求：
- 语气：专业但友好
- 长度：150 字左右
- 包含：会议目的、时间地点、准备要求、RSVP 截止日期
```

### 练习 2: 创建模板

为你的常用任务创建提示词模板：
```
# 代码审查模板
你是资深代码审查员，请审查以下代码：

[代码]

审查要点:
1. 代码规范
2. 潜在 bug
3. 性能问题
4. 安全漏洞
5. 改进建议

输出格式：问题列表 + 优先级 + 修改建议
```

## 💡 高级用法

### 1. 元提示词

让 AI 帮你写提示词：
```
我想让 AI 帮我分析销售数据，请帮我设计一个最好的提示词
```

### 2. 自我验证

```
请回答后，检查你的答案是否满足所有要求
```

### 3. 多轮对话

```
第一轮：列出大纲
第二轮：展开第 1 点
第三轮：补充示例
第四轮：优化语言
```
