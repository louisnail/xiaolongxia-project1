# ⚙️ 配置详解 - 完全掌握 OpenClaw

> 从入门到精通的配置指南

## 📁 配置文件结构

```
openclaw/
├── config.json          # 主配置文件
├── config.local.json    # 本地覆盖（不提交 git）
├── .env                 # 环境变量
└── skills/              # 技能目录
```

## 🔑 核心配置项

### 1. Gateway 配置

```json
{
  "gateway": {
    "port": 3000,              // 监听端口
    "host": "0.0.0.0",         // 监听地址
    "logLevel": "info",        // 日志级别
    "maxConnections": 100,     // 最大连接数
    "timeout": 30000           // 超时时间 (ms)
  }
}
```

**配置说明**:
| 参数 | 默认值 | 说明 |
|------|--------|------|
| port | 3000 | 服务端口 |
| host | 0.0.0.0 | 监听所有网卡 |
| logLevel | info | debug/info/warn/error |
| maxConnections | 100 | 并发连接限制 |
| timeout | 30000 | 请求超时 |

### 2. 渠道配置

#### Telegram
```json
{
  "channels": [
    {
      "type": "telegram",
      "token": "YOUR_BOT_TOKEN",
      "name": "MyBot",
      "enabled": true,
      "config": {
        "pollingInterval": 1000,
        "maxRetries": 3
      }
    }
  ]
}
```

**获取 Token**:
1. 打开 Telegram，搜索 @BotFather
2. 发送 `/newbot`
3. 按提示设置名称
4. 复制 Token

### 3. 模型配置

```json
{
  "models": {
    "default": "qwen-plus",
    "models": [
      {
        "id": "qwen-plus",
        "provider": "dashscope",
        "apiKey": "YOUR_API_KEY",
        "maxTokens": 4096,
        "temperature": 0.7
      },
      {
        "id": "gpt-4",
        "provider": "openai",
        "apiKey": "YOUR_API_KEY",
        "maxTokens": 8192,
        "temperature": 0.7
      }
    ]
  }
}
```

**模型参数说明**:
| 参数 | 说明 | 推荐值 |
|------|------|--------|
| temperature | 创造性 (0-1) | 0.7 |
| maxTokens | 最大输出长度 | 4096 |
| topP | 采样概率 | 0.9 |
| frequencyPenalty | 重复惩罚 | 0.5 |

## 🔐 环境变量管理

### 创建 .env 文件

```bash
# .env
# 不要提交到 git!

# API Keys
OPENAI_API_KEY=sk-xxx
DASHSCOPE_API_KEY=sk-xxx

# Bot Tokens
TELEGRAM_BOT_TOKEN=xxx:xxx
DISCORD_BOT_TOKEN=xxx

# Database
DATABASE_URL=postgresql://user:pass@localhost/db
```

### 在配置中引用

```json
{
  "models": [
    {
      "id": "gpt-4",
      "provider": "openai",
      "apiKey": "${OPENAI_API_KEY}"
    }
  ]
}
```

## 🐛 配置问题排查

### 问题 1: 配置不生效
```
解决步骤:
1. 检查配置文件路径
2. 重启 Gateway
3. 清除缓存
4. 查看日志
```

### 问题 2: 环境变量未加载
```
检查:
1. .env 文件是否存在
2. 是否使用 dotenv 加载
3. 变量名是否正确
4. 是否需要重启
```

### 问题 3: 渠道连接失败
```
排查:
1. Token/密钥是否正确
2. 网络是否通畅
3. 权限是否足够
4. 查看渠道日志
```

## 💡 最佳实践

### 1. 配置分离
```
config.json         # 基础配置（可提交）
config.local.json   # 本地配置（不提交）
.env                # 敏感信息（不提交）
```

### 2. 配置验证
```javascript
// 启动前验证
function validateConfig(config) {
  const errors = [];
  
  if (!config.gateway.port) {
    errors.push('缺少 gateway.port');
  }
  
  if (config.channels.length === 0) {
    errors.push('至少配置一个渠道');
  }
  
  return errors;
}
```

### 3. 配置备份
```bash
# 定期备份配置
cp config.json config.json.backup.$(date +%Y%m%d)
```

### 4. 版本控制
```bash
# git 忽略敏感文件
echo "config.local.json" >> .gitignore
echo ".env" >> .gitignore
```
