## 架构设计

### 1. 模块化设计

**推荐结构**:
```
workspace/
├── config/              # 配置文件
├── skills/              # 自定义技能
├── scripts/             # 工具脚本
├── memory/              # 记忆文件
├── logs/                # 日志目录
└── docs/                # 文档
```

**好处**:
- ✅ 清晰的文件组织
- ✅ 易于维护和扩展
- ✅ 方便版本控制

### 2. 会话管理

**主会话 vs 隔离会话**:

| 场景 | 推荐 | 原因 |
|------|------|------|
| 日常对话 | Main | 保持上下文 |
| 长任务 | Isolated | 避免污染主上下文 |
| 定时任务 | Isolated | 独立执行 |
| 敏感操作 | Isolated | 权限隔离 |

**示例**:
```javascript
// 简单任务 - 主会话
sessions_send({
  message: "今天天气如何？"
});

// 复杂任务 - 隔离会话
sessions_spawn({
  task: "分析这份 100 页的报告",
  cleanup: "delete"  // 完成后清理
});
```

### 3. 记忆系统设计

**三层记忆**:
```
L1 - WORKING (工作记忆)
└── 当前任务上下文
    └── 任务完成即清理

L2 - SHORT_TERM (短期记忆)
└── memory/YYYY-MM-DD.md
    └── 每日记录，保留 7-30 天

L3 - LONG_TERM (长期记忆)
└── MEMORY.md
    └── 重要信息，永久保存
```

**使用建议**:
- 临时信息 → 对话中处理
- 日常记录 → 写入日期文件
- 关键信息 → 更新 MEMORY.md

## 开发规范

### 1. Skills 开发

**命名规范**:
```javascript
// ✅ 好
kebab-case: stock-watcher, file-processor
descriptive: weather-forecast, email-sender

// ❌ 避免
camelCase: stockWatcher
vague: tool1, helper, utils
```

**代码结构**:
```javascript
// 1. 导入依赖
import { fetch } from 'undici';

// 2. 常量定义
const API_URL = 'https://api.example.com';

// 3. 辅助函数
function validateInput(input) { ... }

// 4. 主函数
export async function mainFunction(params) { ... }

// 5. 导出
export { helperFunction };
```

**错误处理**:
```javascript
// ✅ 推荐
try {
  const result = await doSomething();
  return { success: true, data: result };
} catch (error) {
  console.error('[Skill] 执行失败:', error);
  return { 
    success: false, 
    error: '友好的错误信息' 
  };
}

// ❌ 避免
const result = await doSomething(); // 可能抛出异常
return result;
```

## 性能优化

### 1. Token 优化

**策略回顾**:
```
1. 提示词精简 → 省 30-50%
2. 上下文管理 → 省 20-40%
3. 响应控制 → 省 10-30%
4. 缓存复用 → 省 40-60%
```

**实用技巧**:
```javascript
// 只加载必要内容
const snippet = await memory_get({
  path: 'MEMORY.md',
  from: 10,
  lines: 5  // 只读 5 行
});

// 语义搜索代替全文加载
const results = await memory_search({
  query: '股票分析',
  maxResults: 3  // 只取 top3
});
```

### 2. 响应时间优化

**异步处理**:
```javascript
// ✅ 并行执行
const [data1, data2, data3] = await Promise.all([
  fetchAPI1(),
  fetchAPI2(),
  fetchAPI3()
]);

// ❌ 串行执行
const data1 = await fetchAPI1();
const data2 = await fetchAPI2();
const data3 = await fetchAPI3();
```

**超时控制**:
```javascript
// 设置合理超时
const result = await Promise.race([
  doTask(),
  timeout(30000)  // 30 秒超时
]);
```

### 3. 内存管理

**定期清理**:
```javascript
// 清理旧会话
setInterval(async () => {
  const sessions = await sessions_list();
  for (const session of sessions) {
    if (session.lastActive < Date.now() - 3600000) {
      await closeSession(session.id);
    }
  }
}, 300000); // 每 5 分钟
```

## 安全建议

### 1. 密钥管理

**环境变量**:
```bash
# .env (不提交到 git)
API_KEY=sk-xxx
BOT_TOKEN=xxx:xxx
```

**配置引用**:
```json
{
  "models": [{
    "apiKey": "${API_KEY}"
  }]
}
```

### 2. 权限控制

**最小权限原则**:
```javascript
// ✅ 只读权限
const data = await readDatabase({
  table: 'users',
  columns: ['name', 'email'],
  readonly: true
});

// ❌ 过度权限
const data = await executeSQL('SELECT * FROM users');
```

### 3. 输入验证

```javascript
// 验证用户输入
function validateInput(input) {
  if (!input || typeof input !== 'string') {
    throw new Error('无效输入');
  }
  
  if (input.length > 1000) {
    throw new Error('输入过长');
  }
  
  // 防止注入
  const sanitized = input.replace(/[<>]/g, '');
  return sanitized;
}
```

## 运维部署

### 1. 生产环境部署

**PM2 配置**:
```javascript
// ecosystem.config.js
module.exports = {
  apps: [{
    name: 'openclaw',
    script: 'npm',
    args: 'start',
    instances: 1,
    exec_mode: 'cluster',
    env: {
      NODE_ENV: 'production'

    error_file: './logs/error.log',
    out_file: './logs/out.log'
  }]
```

**启动命令**:

### 2. 监控告警


## 故障排查

### 1. 问题诊断流程

```
1. 查看日志 → 定位错误
2. 复现问题 → 确认范围
3. 检查配置 → 排除配置问题
4. 查看资源 → CPU/内存/网络
5. 逐步排查 → 隔离问题点
```

### 2. 常见问题速查

**Gateway 无法启动**:
```bash
# 检查端口占用
lsof -i :3000

# 查看日志
tail -f logs/gateway.log

# 验证配置
openclaw config validate
```

**渠道连接失败**:
```bash
# 检查 Token
echo $TELEGRAM_BOT_TOKEN

# 测试网络
curl https://api.telegram.org

# 查看渠道日志
tail -f logs/channels.log
```

**Skills 不工作**:
```bash
# 检查技能路径
ls -la skills/

# 验证 SKILL.md
cat skills/weather/SKILL.md

# 查看技能日志
tail -f logs/skills.log
```

**内存泄漏**:
```bash
# 监控内存
pm2 monit

# 查看堆快照
node --inspect app.js

# 定期重启
pm2 restart openclaw --max-memory 500M
```

### 3. 调试技巧

**启用调试模式**:

**添加断点**:
