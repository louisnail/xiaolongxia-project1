🚀 QoderWork 安装 OpenClaw

利用 QoderWork 桌面 AI 助手的自主规划能力，一键自动化安装 OpenClaw

⏱️ 学习时长：15 分钟  |  ⭐ 难度等级：简单  |  🛠️ 前置要求：已安装 QoderWork Desktop

💡 QoderWork 核心优势

  • 自主规划 - AI 自动分解复杂任务，无需逐步指令
  • 本地执行 - 直接在本地文件系统中操作，无需上传下载
  • 安全沙箱 - 所有操作在本地沙箱中运行，数据不出设备
  • 直接文件访问 - AI 可直接读写、编辑、创建文件

1下载并安装 QoderWork

访问 qoder.com/download 下载 QoderWork Desktop：

  • macOS - 支持 macOS 14+ (Apple Silicon / Intel)
  • Windows - 支持 Windows 10+

安装完成后启动 QoderWork，登录或注册账号。

2创建工作目录

在 QoderWork 中：

  1. 点击 "Open Folder" 或 "新建工作区"
  2. 选择或创建一个空文件夹（例如：~/openclaw-workspace）
  3. QoderWork 会请求访问权限，点击"允许"
💡 提示：QoderWork 需要访问本地文件权限来执行安装操作。

3发送安装指令

在 QoderWork 的 AI 对话框中输入以下指令（复制粘贴）：

🤖 完整安装指令

请帮我安装 OpenClaw（AI 助手框架），利用你的自主规划能力完成以下任务：

## 任务目标
在本地安装并配置 OpenClaw，使其可以通过 Telegram 或微信访问。

## 需要你完成的具体步骤

### 1. 环境检查
- 检查 Node.js 是否已安装（需要 v18+）
- 如果未安装，提供安装指引

### 2. 克隆项目
- 在当前工作目录克隆 OpenClaw 仓库
- 命令：git clone https://github.com/openclaw/openclaw.git
- 进入项目目录

### 3. 安装依赖
- 执行 npm install 安装所有依赖
- 等待安装完成并报告结果

### 4. 创建配置文件
创建以下文件：

**.env 文件** - 环境变量：
```
DASHSCOPE_API_KEY=请替换为你的通义千问 API Key
```

**config.yaml 文件** - 主配置：
```yaml
gateway:
  port: 3000
  host: 0.0.0.0

channels:
  telegram:
    - token: "请替换为你的 Telegram Bot Token"
      name: "OpenClaw Bot"
      model: "qwen-plus"
      thinking: "on"
      systemPrompt: |
        你是一位专业、友好、高效的 AI 助手。
        你擅长用简洁清晰的语言解答问题。

models:
  qwen-plus:
    provider: dashscope
    apiKey: ${DASHSCOPE_API_KEY}
    model: qwen-plus
    maxTokens: 2000

memory:
  enabled: true
  path: ./MEMORY.md
```

### 5. 初始化记忆文件
创建 MEMORY.md 文件，包含基本结构：
```markdown
# MEMORY.md - 长期记忆

## 用户偏好
- 称呼：用户
- 时区：Asia/Shanghai
- 语言：中文

## 项目上下文
- 当前项目：OpenClaw
```

### 6. 启动 OpenClaw
- 执行启动命令：npm start
- 监控启动日志
- 报告 Gateway 是否成功启动

### 7. 验证和测试
- 告诉我如何验证安装成功
- 提供下一步操作指引

## 执行要求
1. 请自主规划执行步骤，不需要我逐步确认
2. 每完成一个步骤请简要报告
3. 遇到错误时详细说明原因和解决方案
4. 所有配置文件创建后请告诉我文件位置
5. 最后提供一个完整的安装总结

请现在开始执行，首先检查环境。

5提供必要信息

QoderWork 在执行过程中可能会询问：

  • 通义千问 API Key - 从 DashScope 控制台 获取
  • Telegram Bot Token - 在 Telegram 中搜索 @BotFather 创建机器人

将这些真实值提供给 QoderWork，它会更新到配置文件中。

🐛 常见问题

Q1: QoderWork 无法访问本地文件

解决方法：在系统设置中授予 QoderWork "完全磁盘访问权限"（macOS）或管理员权限（Windows）。

Q2: Node.js 版本过低

QoderWork 会自动检测并提示升级。可以访问 nodejs.org 下载最新版本。

Q3: Git 未安装

QoderWork 会提供安装指引，或可以手动下载 Git：git-scm.com

Q4: Gateway 启动失败

QoderWork 会自动检查：

  • 端口 3000 是否被占用
  • .env 文件是否存在
  • API Key 是否正确
  • 依赖是否完整安装
