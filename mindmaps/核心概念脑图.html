🗺️ OpenClaw 核心概念脑图

全面理解 OpenClaw 的架构组成、消息流程、记忆系统、定时任务等核心概念

核心概念全景图

8 大核心模块，构建完整的 OpenClaw 知识体系

🏗️ 架构组成

  • Gateway - 核心服务进程，管理所有功能
  • Sessions - 会话管理，保持对话上下文
  • Agents - AI 代理，执行具体任务
  • Skills - 技能扩展，自定义功能
  • Channels - 渠道接入（Telegram/微信/Discord）
  • Memory - 记忆系统（MEMORY.html + memory/）

📬 消息流程

  • 用户发送消息 → 渠道插件接收
  • 渠道插件 → Gateway 处理
  • Gateway → Agent 推理
  • Agent → 调用工具（可选）
  • Agent → 生成回复
  • Gateway → 渠道插件 → 用户

🧠 记忆系统

  • MEMORY.html - 长期记忆（仅主会话加载）
  • memory/ - 日常笔记目录
  • memory_search - 语义搜索记忆
  • memory_get - 精确读取记忆片段
  • session transcripts - 会话记录（可选搜索）
  • 安全隔离（群聊不加载个人记忆）

⏰ 定时任务

  • at - 一次性定时任务（绝对时间）
  • every - 周期性任务（间隔时间）
  • cron - Cron 表达式（灵活调度）
  • systemEvent - 系统事件（主会话）
  • agentTurn - Agent 执行（隔离会话）
  • 应用场景：提醒、日报、周报、自动检查

💰 Token 优化

  • 提示词精简 - 去除冗余，降低 30-50%
  • 上下文管理 - 限制历史消息数量
  • 响应控制 - 使用 NO_REPLY 减少输出
  • 缓存复用 - 避免重复计算
  • 模型选择 - 简单任务用轻量模型
  • 成本降低：30-60%

🛠️ Skills 开发

  • SKILL.html - 技能描述和触发规则
  • index.js - 主逻辑代码
  • package.json - 依赖管理
  • 开发流程 - 需求→设计→编码→调试→测试
  • 调试技巧 - 日志、断点、错误处理
  • 最佳实践 - 代码规范、性能优化

🔗 渠道集成

  • Telegram - Bot API，简单易用
  • 微信 - 个人号/企业微信
  • Discord - 服务器机器人
  • Feishu - 飞书机器人
  • WhatsApp - WhatsApp Business API
  • Slack - Slack Bot

🎯 使用场景

  • 个人助理 - 日程管理、邮件处理
  • 知识管理 - 文档管理、笔记系统
  • 工作自动化 - 数据处理、报告生成
  • 开发辅助 - 代码生成、代码审查
  • 内容创作 - 文章写作、社交媒体
  • 研究分析 - 文献调研、数据分析
